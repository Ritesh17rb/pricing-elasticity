Methodology: Current State KPIs
Data Sources
  • store_channel_week_panel.csv - weekly Yum brand channel orders, sales, margin, and mix
  • store_item_week_panel.csv - weekly Yum brand item demand, price, promo, and elasticity rows
  • promo_calendar.csv - generated promotion pressure and campaign windows used for context
KPI Formulas
Average Order Value (AOV)
Average Check = Total Revenue / Weekly Orders

Calculated from the latest weekly store-channel operating panel

Customer Lifetime Value (LTV)
LTV = AOV × Average Tenure (months) × (1 - Repeat Loss Rate)

Discounted over 24-month horizon at 5% annual rate

Repeat Loss Rate
Monthly Repeat Loss = Lapsed Customers / Active Customers (30-day window)

Smoothed with 90-day moving average

Key Assumptions
  • All data is modeled to reflect a realistic seasonal operating pattern
  • Baseline period: Last 12 weeks of available data
  • Currency: USD
  • No seasonal adjustments applied
Methodology: Traffic Acquisition Elasticity
Elasticity Model

Traffic acquisition elasticity measures how weekly orders in the selected Yum brand respond to effective price changes by channel and visit mission.

Core Formula
% Change in Acquisitions = Elasticity Coefficient × % Change in Price

Example: If elasticity = -2.1 and price increases by 10%, acquisitions decrease by 21%

Elasticity Coefficients (Industry Benchmarks)
Order Channel Base Elasticity Source
Drive-Thru -1.7 Largest base volume, lower migration risk
Pickup / App -2.0 Higher digital responsiveness and promo sensitivity
Adjustment Factors

Base elasticity is modified by:

  • Tenure: New users (0-3 months) are 1.3× more sensitive
  • Age Group: 18-24 demographic is 1.2× more sensitive
  • Acquisition Channel: Paid ads users are 1.4× more sensitive
  • Time Horizon: Long-term effects (12+ months) are 1.5× stronger
Scenario Simulation
  1. Calculate price change percentage: (New Price - Current Price) / Current Price
  2. Apply elasticity: Demand Change = Elasticity × Price Change%
  3. Project new acquisitions: New Acq = Baseline Acq × (1 + Demand Change)
  4. Calculate revenue impact: New Rev = New Acq × New Price × Retention Rate
ML Model: Poisson GLM (Python)
Model Implementation python/acquisition_model.py

Algorithm: Poisson Generalized Linear Model

Execution: Runs in browser via Pyodide (Python + NumPy in WebAssembly)

Link Function: log(E[adds]) = β₀ + β₁×price + β₂×discount + β₃×is_promo + β₄×segment

Pre-fitted Coefficients:

  • intercept = 6.5 → baseline ~665 adds/month
  • price = -0.25 → price sensitivity
  • promo_discount = 0.03 → discount effectiveness
  • is_promo = 0.4 → promo boost multiplier
  • segment_elasticity_factor = 0.12 → segment adjustment

Why Poisson? Customer counts are whole numbers (0, 1, 2...). This statistical method naturally fits count data and auto-adjusts as volumes change.

Important: Model coefficients are industry benchmarks and should be retrained on business data before deployment.
Methodology: Repeat-Visit Loss Elasticity
Repeat Loss Model

Repeat-visit loss elasticity models how Taco Bell guests trim frequency over time after menu price increases or value support roll-off.

Repeat Loss Formula
Additional Repeat-Visit Loss% = Repeat-Visit Loss Elasticity × Price Change% × Time Factor

Example: 10% price increase with -1.8 elasticity → +18% repeat loss (baseline 3% → 3.54%)

Time-Lagged Effects

Repeat loss doesn't happen instantly. We model temporal dynamics:

Period Effect Multiplier Explanation
0-1 month 0.3× Immediate reaction (angry users)
2-3 months 1.0× Peak repeat loss period
4-6 months 0.7× Tapering off (remaining users adjusted)
7-12 months 0.4× Long-tail effect
Cohort-Specific Repeat Loss

Different customer segments have different repeat-loss sensitivities:

  • New customers (0-6 months): 2.5× baseline repeat-loss elasticity
  • Mid-tenure (6-18 months): 1.0× baseline
  • Loyal (18+ months): 0.5× baseline (more forgiving)
  • Bundle customers: 0.3× baseline (highest repeat rate)
Retention Tactics

Model includes optional retention strategies:

  • Win-back campaigns: Can reduce repeat loss by 15-25%
  • Price protection: Exempt loyal customers from increases
  • Loyalty offers: Offer 20% off for long-tenure customers
ML Model: Logistic Regression (Python)
Model Implementation python/churn_model.py

Algorithm: Logistic Regression with time-lagged effects

Execution: Runs in browser via Pyodide (Python + NumPy)

Link Function: logit(repeat_loss) = β₀ + β₁×price_change + β₂×time_interaction

Pre-fitted Coefficients:

  • intercept = -2.944 → baseline 5% repeat-loss probability
  • price_change_pct = 0.01 → base price effect
  • price_x_0_4wks = 0.006 → immediate effect multiplier
  • price_x_4_8wks = 0.018 → roll-off peak multiplier
  • price_x_8_12wks = 0.028 → peak repeat-loss multiplier
  • price_x_12plus = 0.008 → long-tail stabilization

Why Logistic? Binary outcome (repeat loss yes/no) requires logit link. Time-lagged interactions capture delayed repeat-loss response to price changes.

Data Source: Repeat-visit patterns calibrated from Taco Bell weekly item and channel panels plus industry research.
Methodology: Order Channel Migration
Migration Model

Models Taco Bell guest movement between Drive-Thru and Pickup / App when relative check gaps change, with spillover into other channels.

Migration Formula
Migration Rate = Base Rate × Price Gap Factor × Elasticity

Where Price Gap = (Higher Channel Price - Lower Channel Price) / Lower Tier Price

Migration Patterns
Fallback Triggers (Pickup / App → Drive-Thru)
  • Base downgrade rate: 2-5% monthly
  • Increases by 1.5% for every $1 price gap increase
  • Peak downgrade occurs 2-3 months after price change
Owned-Channel Migration Triggers (Drive-Thru → Pickup / App)
  • Base upgrade rate: 3-7% monthly
  • Decreases by 0.8% for every $1 price gap increase
  • Motivated by premium assortment and exclusive sets
Simulation Logic
  1. Calculate new price gaps between all tier pairs
  2. Estimate migration probabilities using logistic regression
  3. Apply cohort-specific modifiers (tenure, engagement)
  4. Simulate month-by-month transitions using Markov chain
  5. Calculate net revenue impact accounting for mix shift
Elasticity Cross-Effects

Migration is influenced by relative elasticities:

Cross-Elasticity = -0.5 × (Channel A Elasticity - Channel B Elasticity)

Higher elasticity differential = more migration

ML Model: Multinomial Logit (Python)
Model Implementation python/migration_model.py

Algorithm: Multinomial Logistic Regression (Softmax)

Execution: Runs in browser via Pyodide (Python + NumPy)

Outputs: Migration probability matrix across all tier pairs

Supported Channel Structures:

  • 2-Group: Drive-Thru ↔ Pickup / App
  • 3-Group Owned: Drive-Thru, Pickup / App, Delivery
  • 3-Group Broader: Drive-Thru, Pickup / App, Other Channels

Sample Coefficients (3-Tier Bundle):

  • ad_free_to_bundle intercept = -0.5 → moderate upgrade appeal
  • value_savings_pct = 0.03 → 21% savings → +0.63 utility
  • bundle_need = 0.8 → Value sets drive 55-70% conversion
  • tenure_months = 0.012 → loyal customers more likely to upgrade

Why Multinomial Logit? Multiple discrete channel choices (stay, migrate, leak). Softmax ensures probabilities sum to 1 and handles asymmetric channel cross-effects.

Calibration: Migration rates calibrated from Taco Bell store-channel history and relative check gaps between Drive-Thru and Pickup / App.
Methodology: Customer Cohorts
Segmentation Framework

3-axis behavioral segmentation framework creating 250 unique segments (5 × 5 × 5 × 2 channel groups).

Three Segmentation Axes
Axis 1: Basket Value & Mix (5 segments)
  • Single-SKU Staple: Repeat purchases of a hero SKU
  • Multi-SKU Builder: Builds larger baskets
  • Value Bundle Buyer: Prefers sets and value packs
  • Premium Add-On: Adds premium or limited SKUs
  • Trial-Size Sampler: Tests mini or entry packs
Axis 2: Repeat Behavior (5 segments)
  • Prestige Loyalist: High repeat rate in prestige channel
  • Value Seeker: Responds to moderate discounts
  • Deal Hunter: Promo-first, high drop-off risk
  • Occasional Shopper: Low frequency purchases
  • Channel Switcher: Moves between mass and prestige
Axis 3: Acquisition Driver (5 segments)
  • Seasonal First-Time: First purchase driven by season
  • Routine Refill: Replenishment driven
  • Gift Buyer: Purchases tied to gifting
  • Influencer Discovered: Social-led discovery
  • Promo Triggered: Enters only with discounts
Segment Elasticity Calculation
Segment Elasticity = Base Tier Elasticity × Axis1 Modifier × Axis2 Modifier × Axis3 Modifier

Example: Drive-Thru deal-seeker, app-migration prone, value-box buyer:
-2.1 (base) × 1.3 (promo) × 1.2 (deal hunter) × 1.1 (bundle) = -3.60 (highly elastic)

Data Sources
  • segments.csv: 250 cohort definitions with characteristics
  • segment_kpis.csv: Historical performance by cohort
  • segment_elasticity.json: Calibrated elasticity parameters
Validation: Segments validated against holdout data showing 85% accuracy in predicting repeat-loss behavior.
Methodology: Segment Elasticity Comparison
Comparative Analysis

Compares price sensitivity across customer segments to identify optimal pricing strategies.

Analysis Methods
1. Elasticity Heatmap

Visual matrix showing elasticity coefficients across segments.

  • Color scale: Green (inelastic -0.5 to -1.5) → Red (highly elastic -3.0+)
  • Sorting: By elasticity magnitude or revenue contribution
  • Filtering: By tier, acquisition channel, engagement level
2. Scatter Plot Analysis

2D visualization: Elasticity (X-axis) vs Revenue Impact (Y-axis)

  • Quadrant I: High elasticity, high revenue → Risky to increase prices
  • Quadrant II: Low elasticity, high revenue → Safe to increase prices
  • Quadrant III: Low elasticity, low revenue → Low priority
  • Quadrant IV: High elasticity, low revenue → Discount opportunity
3. 3D Radial Chart

Multi-dimensional view plotting segments across three axes:

  • Radius: Elasticity magnitude
  • Angle: Revenue contribution
  • Color: Repeat-loss risk level
Filtering & Aggregation

Analysis supports multiple filter combinations:

  • By Menu Ladder: Compare within Entry & Value or Core & Premium ladders
  • By Tenure: New (0-6mo), mid-tenure (6-18mo), loyal (18+mo)
  • By Engagement: Heavy users (>20hr/month), light users (<5hr/month)
  • By Risk: High repeat-loss risk (>10% monthly), stable (<3% monthly)
Export Capabilities
  • CSV Export: Full cohort data with elasticity and KPI metrics
  • SVG Export: Vector graphics for presentations
  • JSON Export: Raw data for further analysis
Key Insights
  • Segments with elasticity < -2.5 are highly price-sensitive
  • Segments with elasticity > -1.0 are inelastic (price increase opportunity)
  • Focus price optimization on high-revenue, low-elasticity segments
  • Consider targeted discounts for high-elasticity, low-engagement segments
